한 줄 요약 (TL;DR)

“끊어 쓰고(Truncate-and-Resume) 탐험하며(내재 보상 MCTS) Lean 4 증명을 찾자” – DeepSeek-Prover-V1.5는 7 B 모델로 miniF2F 63.5 % / *ProofNet 25.3 %*를 달성해 이전 SOTA를 +9.0 pp / +7.2 pp 앞지르면서도 탐색 샘플을 최대 64 × 절감했다.

핵심 아이디어

  • Whole-proof의 단순·빠른 한 방 생성과

  • Step-wise 탐색의 세밀한 피드백을

  • **“Truncate-and-Resume + RMaxTS”**라는 하이브리드 프레임으로 융합

    • 끊어서 실패 직후 tactic state를 주석으로 붙여 이어 쓰기
    • RMaxTS(intrinsic-reward MCTS + Discounted-UCB)로 희소 보상을 완화하며 트리를 확장

배경 – 그들이 해결한 문제

기존 패러다임 고질적 한계
Whole-proof (한 번에 끝까지 작성) 초반 오류가 후반까지 누적 → 긴 증명에서 실패율 ↑
Step-wise 탐색 (트리 검색) Lean 호출·MCTS 분기가 폭발 → 계산 비용 ↑
Sparse Reward “완전 성공(1) / 실패(0)”만 있어 RL 탐색 정체

결국 **“중간 피드백을 받으면서도 비용은 낮추는 길이 없을까?”**가 핵심 연구 공백이었다.

새로운 접근법 – DeepSeek-Prover-V1.5

  1. Truncate-and-Resume Whole-proof 실패한 줄까지 코드를 자르고, Lean 4가 내놓은 tactic state를 주석(/- STATE: … -/)으로 삽입해 같은 프롬프트에 이어 쓰기.

  2. RMaxTS (MCTS with Intrinsic Reward & Discounted-UCB)

    • 새 노드(미탐색 tactic state) 발견 시 R_int = 1
    • 보상·방문수를 γ = 0.99로 감쇠한 DUCB로 분기 선택 → 최신 정보를 우선 반영

  3. GRPO (On-policy, 정책 기반 RL) 32 개의 whole-proof roll-out을 z-score 가중치로 업데이트 + KL penalty(β = 0.02)로 안정화.

  1. Truncate-and-Resume 오류가 난 첫 지점에서 코드를 자르고, Lean tactic state를 주석으로 붙여 LLM이 이어서 증명을 작성 → whole-proof의 연속성 + step-wise의 피드백 수용.

    • 수식·코드 데이터 재사전학습

4. 왜 잘 작동하나? (직관)

  1. 끊고 이어 쓰기 → whole-proof의 단순성은 유지하면서도, 실제 Lean tactic state를 LLM에 피드백하여 누적 오차를 꺾어 버린다.
  2. RMaxTS → “새 상태일 때만 1점”이라는 보상 덕에 트리 폭을 넓히는 방향으로 유도, sparse-reward 문제 완화.

Discounted-UCB (Q_DUCB) 최근 보상을 크게 반영해 탐험 보너스 조절 (탐험↔활용 균형) DUCB 식 (7) with γ=0.99
GRPO KL penalty (β = 0.02) 정책 폭발 방지·활용 촉진 학습 세팅
샘플 그룹 정규화 (w_i z-score) 동일 배치(32개) 내 상대 성과에 따라 gradient 가중 GRPO 설명

별도 ε-greedy·entropy reg는 사용하지 않으며, DUCB + KL 조합이 탐험/활용 트레이드오프의 핵심이다.

3. 학습 패러다임 요약

구분 설명
On-/Off-policy 온-폴리시 — 매 스텝 현행 πθ로 32 개 proof를 새로 roll-out
알고리즘 유형 정책-기반 (REINFORCE 계열) · critic 없이 GRPO로 업데이트
핵심 업데이트
\(L(θ)=\sum_i w_i \log π_θ(y_i
 x) - β,\text{KL}[π_θ‖π_\text{SFT}]\) where w_i는 그룹 z-score
Replay buffer 사용하지 않음 — on-policy & sparse reward 구조상 즉시 폐기

🔑 이해 포인트

  • 탐험은 _새로운 tactic state_에 즉시 1점을 주는 RMaxTS와 γ-감쇠 UCB로 달성.
  • 활용은 DUCB의 평균 가치 항(term 1) + GRPO의 KL 규제로 자연스럽게 확보.
  • On-policy이므로 리플레이 버퍼는 불필요하며 Lean 4가 곧 시뮬레이션 환경 역할을 한다.
